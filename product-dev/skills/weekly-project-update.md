Prepare a concise weekly project update.

## Goal
Produce a quick, clear update for stakeholders without unnecessary fluff.

## Input
The project status, changes since the last update, current risks, and next steps.

## Procedure
1. Write down the main progress since the last update.
2. Highlight completed items and significant changes.
3. Identify current risks, blockers, and missing decisions.
4. Propose the nearest next steps.

## Output
Use this structure:
### Progress
### Risks
### Next steps

## Rules
- Write concisely, without a marketing tone.
- Highlight changes, blockers, and decisions.
- If the data is insufficient, state it explicitly.
